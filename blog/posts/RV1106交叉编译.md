# RV1106交叉编译

> 整理自Luckfox官方文档，适用于RV1106，记录交叉编译环境配置及应用程序编译流程。


## 1. 安装交叉编译工具链

Luckfox 提供两套工具链，分别对应 **Buildroot（uclibc）** 和 **Ubuntu（glibc）** 系统。

| 目标系统 | 工具链说明 | 下载地址 |
|---------|-----------|---------|
| **Buildroot** | uclibc 工具链，集成在 SDK 内 | [arm-rockchip830-linux-uclibcgnueabihf.tar.gz](https://files.luckfox.com/wiki/Luckfox-Pico/Software/arm-rockchip830-linux-uclibcgnueabihf.tar.gz) |
| **Ubuntu** | glibc 工具链，需从 ARM 官网获取 | [gcc-arm-11.2-2022.02-x86_64-arm-none-linux-gnueabihf.tar.gz](https://files.luckfox.com/wiki/Luckfox-Pico/Software/gcc-arm-11.2-2022.02-x86_64-arm-none-linux-gnueabihf.tar.gz) |

### 解压到用户目录（示例）

```bash
# 解压 uclibc 工具链（Buildroot 用）
tar zxvf arm-rockchip830-linux-uclibcgnueabihf.tar.gz -C ~/

# 解压 glibc 工具链（Ubuntu 用）
tar zxvf gcc-arm-11.2-2022.02-x86_64-arm-none-linux-gnueabihf.tar.gz -C ~/
```

---

## 2. 编译 Buildroot 系统下的应用程序

> 本节适用于开发板运行 **Buildroot** 系统，使用 uclibc 工具链。

### 方式一：通过环境变量（全局生效）

编辑 `~/.bashrc` 或 `~/.profile`，添加以下内容：

```bash
export ARCH=arm
export CROSS_COMPILE=arm-rockchip830-linux-uclibcgnueabihf-
export PATH=/home/你的用户名/luckfox-pico/tools/linux/toolchain/arm-rockchip830-linux-uclibcgnueabihf/bin:$PATH
```

使配置生效：

```bash
source ~/.bashrc   # 或 source ~/.profile
```

验证：

```bash
echo $ARCH
echo $CROSS_COMPILE
arm-rockchip830-linux-uclibcgnueabihf-gcc --version
```

编译示例程序：

```bash
arm-rockchip830-linux-uclibcgnueabihf-gcc hello.c -o hello
```

### 方式二：通过 Makefile（项目局部）

在与 `hello.c` 同级目录下创建 `Makefile`：

```makefile
CC := /home/你的用户名/luckfox-pico/tools/linux/toolchain/arm-rockchip830-linux-uclibcgnueabihf/bin/arm-rockchip830-linux-uclibcgnueabihf-gcc
CFLAGS := -Wall -O2

hello: hello.c
	$(CC) $(CFLAGS) $^ -o $@

clean:
	rm -f hello
```

执行编译：

```bash
make
```

> **注意**：Makefile 中的命令前必须使用 **Tab** 缩进，不可用空格。

## 3. 编译 Ubuntu 系统下的应用程序

> 本节适用于开发板运行 **Ubuntu** 系统，使用 glibc 工具链。

### 方式一：通过环境变量（全局生效）

编辑 `~/.bashrc` 或 `~/.profile`，添加：

```bash
export ARCH=arm
export CROSS_COMPILE=arm-none-linux-gnueabihf-
export PATH=/home/你的用户名/gcc-arm-11.2-2022.02-x86_64-arm-none-linux-gnueabihf/bin:$PATH
```

使配置生效并验证：

```bash
source ~/.bashrc
arm-none-linux-gnueabihf-gcc --version
```

编译：

```bash
arm-none-linux-gnueabihf-gcc hello.c -o hello
```

### 方式二：通过 Makefile（项目局部）

创建 `Makefile`（与 `hello.c` 同目录）：

```makefile
CC := /home/你的用户名/gcc-arm-11.2-2022.02-x86_64-arm-none-linux-gnueabihf/bin/arm-none-linux-gnueabihf-gcc
CFLAGS := -Wall -O2

hello: hello.c
	$(CC) $(CFLAGS) $^ -o $@

clean:
	rm -f hello
```

执行编译：

```bash
make
```

---

## 4. 运行程序

将生成的可执行文件 `hello` 通过 ADB、SCP 或 U 盘等方式传输到开发板，添加执行权限并运行：

```bash
chmod +x hello
./hello
```

---

## 注意事项

- 环境变量设置中，`PATH` 添加时建议将工具链路径置于 `$PATH` **之前**，确保优先调用。
- 若使用 `~/.profile`，修改后可能需要重新登录或手动 `source`。
- 两种工具链（uclibc / glibc）**不可混用**，请根据开发板实际系统选择。
- Makefile 中的路径请替换为实际 SDK 解压路径。
- 若编译时提示找不到头文件或库，可检查工具链的 `sysroot` 是否正确。
